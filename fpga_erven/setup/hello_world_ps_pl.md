
| Register | Offset | Description |

## 6. Software Application

**Platform:** `zcu106_platform` (standalone, psu_cortexa53_0)  
**Application:** `gpio_leds` (Empty Application C)

### 6.1 Source: `main.c`

```c
#include "xil_io.h"
#include "xparameters.h"

/* AXI GPIO base address (from xparameters.h) */
#define GPIO_BASEADDR   XPAR_XGPIO_0_BASEADDR   /* 0xA0000000 */

/* AXI GPIO register offsets */
#define GPIO_DATA_REG   0x00   /* data register  */
#define GPIO_TRI_REG    0x04   /* direction reg  */

int main(void)
{
    /* Set all 8 GPIO pins as outputs (0 = output) */
    Xil_Out32(GPIO_BASEADDR + GPIO_TRI_REG,  0x00000000U);

    /* Drive all 8 LEDs high */
    Xil_Out32(GPIO_BASEADDR + GPIO_DATA_REG, 0x000000FFU);

    while (1);   /* keep state */
    return 0;
}
```

### 6.2 Notes on Vitis 2025.1 / SDT

Vitis 2025.1 uses the System Device Tree (SDT) flow instead of the classic
`xparameters.h` device-ID approach. As a result:

- `XPAR_AXI_GPIO_0_DEVICE_ID` is **not** generated in SDT mode.
- `XGpio_Initialize()` with a device ID does **not** compile.
- The correct approach is to use `Xil_Out32` / `Xil_In32` with the base
  address macro `XPAR_XGPIO_0_BASEADDR` from `xparameters.h`.

## 7. Programming Sequence

```
1. Open Vivado → Generate Bitstream
2. File → Export Hardware (include bitstream) → .xsa file
3. Tools → Launch Vitis IDE
4. Create Platform Project from .xsa (standalone, psu_cortexa53_0)
5. Create Application Project (Empty Application C) linked to platform
6. Add main.c with Xil_Out32 code above
7. Build application
8. Run → Launch on Hardware (JTAG)
```

## 9. Known Issues and Workarounds

| Issue | Root cause | Workaround |
|-------|-----------|------------|
| `XPAR_AXI_GPIO_0_DEVICE_ID` undeclared | SDT flow does not generate device-ID macros | Use `XPAR_XGPIO_0_BASEADDR` with `Xil_Out32` directly |
| `BD 41-758` clock warning (maxihpm1_fpd_aclk) | HPM1 FPD port enabled but unused | Disable HPM1 FPD in PS configuration |
| `BD 41-237` top module not found | HDL wrapper not created | Right-click .bd → Create HDL Wrapper → Let Vivado manage |
